Аптека (bddartapteka) — модульное CLI-приложение на Dart
Консольное приложение для учета товаров, поставщиков и персонала аптеки с хранением данных в SQLite.

Предметная область
Система автоматизации аптеки включает 5 сущностей:

товары (Product);

поставщики (Supplier);

персонал (Employee);

договоры поставки (SupplyContract) с привязкой к поставщику;

журнал смен (ShiftLog) с привязкой к сотруднику.

Архитектура и структура папок
Проект разделен на модули по слоям:

Plaintext
lib/
  src/
    domain/                # сущности и бизнес-логика (валидация)
      models/
        product.dart
        supplier.dart
        employee.dart
        supply_contract.dart
        shift_log.dart
      validators/
        validators.dart
    data/                  # SQLite и SQL-логика
      database.dart
      repositories/
        product_repository.dart
        supplier_repository.dart
        employee_repository.dart
        contract_repository.dart
        shift_log_repository.dart
    cli/                   # меню и обработка ввода
      menu.dart
      input_helper.dart
bin/
  main.dart                # точка входа (main)
test/
  domain_test.dart         # тесты сериализации
  validation_test.dart     # тесты бизнес-логики
  data_test.dart           # тесты репозиториев
Что вынесено в каждый слой и почему
domain: описывает данные и преобразования (toMap/fromMap), а также правила валидации. Не зависит от SQL и консоли.
data: отечает за хранение, создание таблиц (включая связи Foreign Key) и CRUD-запросы.
cli: отвечает за взаимодействие с пользователем (stdin/stdout) и навигацию по меню.
bin: только запуск приложения, настройка кодировки UTF-8 и закрытие ресурсов.
Такое разделение упрощает поддержку, тестирование и замену инфраструктуры (например, SQLite на другой способ хранения или CLI на графический интерфейс).

Реализованная функциональность
просмотр списков товаров, поставщиков, сотрудников, договоров и смен;
добавление товаров, поставщиков и персонала;
оформление договоров поставки и регистрация смен (с демонстрацией связи FOREIGN KEY);
вывод всех таблиц БД в консоль общим списком.

Валидация
Реализованы проверки:
обязательный текстовый ввод: значение не пустое после trim();
числовые поля для товаров: цена (double) и количество (int) должны быть корректными числами больше 0.

Тесты
В проекте есть 3 обязательных типа тестов:
toMap/fromMap для сущности Product (проверка конвертации модели);
вставка, чтение и удаление записи из временной базы данных в оперативной памяти (sqlite3.openInMemory());
проверка валидации (обработка пустых строк и отрицательных значений).

Запуск
Установить зависимости:
dart pub get

Запустить приложение:
dart run bin/main.dart

Запустить тесты:
dart test
